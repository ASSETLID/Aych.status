Weather Page

The Opam Weather Service (ows) runs on regular intervals an evaluation of the solvability of dependencies in the official OPAM repository, for {{switches|length}} stable releases of OCaml.

The weather page provides an overview of all available packages in the opam repository. A package can have multiple versions available in the opam repository, and each version can be available or not for a specific OCaml release.

Each package is categorized as follows :

  • A package is Broken if it is broken for all (declared) Switch
  • A package is Partially Broken ( if it is broken on some (declared) Switch
  • A package is Correct if it is installable on All (declared) Switch

For each ocaml release, we provide a quick snapshot of the status of all versions of this package avalaible for a particular release.

  • means that a package cannot be used with a specific version of the OCaml compiler because all available versions are not installable
  • means that a package can be used, but some versions are not installable
  • means that all versions are installable
  • : means that there are no versions of this package available for this OCaml release

Packages are ordered in the main table considering the percentage of versions available versus the number of not installable versions.

Weather symbols are associated to each package.

  • A sunny symbol means all versions are installable
  • A cloudy symbol means that between 0% to 20% of the versions available for this package are not installable
  • A stormy symbol means that between 20% to 100% of the versions available for this package are not installable

When more then one symbol is present, the first one refers to the status of the package in the past, and the second one refers to the present status of the package.

The hyperlink assiociated to each package name, open a modal window providing an overview of all versions of the package, for all OCaml releases. The modal view contains, for each version a symbol highlighting the status of the package, the symbol to inspect the reasons causing this specific version to be not installable and the symbol to display the explanation graph. Cliking on the symbols opens a new tab providing a human readable explanation. Cliking on the symbols opens a new tab providing the graphical representation of the explanation.

Dashboard Page

The Dashboard page provides an index of packages using the maintainer email as groupping key. For each maintainer, we show the status of all his/her packages as presented on the main page. An RSS feed is also available from the same page.

Summary Page

The summary page provides a list of packages that are the reason why other packages cannot be installed. For each tuple, (package, version) and for each OCaml release we give the aggregate number of package versions that are not installable.

The hyperlink assiociated to each package name, open a modal window providing an overview of all packages broken by this package.

Backlog Page

The Backlog page provides a list of the last 10 previous runs of the ows service, one for each commit. All previous runs are kept in a compress archive and available for download
